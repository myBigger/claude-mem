# PrisonSIS 安全性说明

> 解答一个最根本的问题：这份笔录，是真的吗？

---

## 一、先说结论，再说原因

**结论：**
在 PrisonSIS 里，一份"已签名"的笔录，其内容无法被任何人篡改——包括程序员、管理员、任何政府部门，除非同时拥有你的 PIN 码和你的电脑。

**这不是承诺，这是数学保证。**

下面解释原因。

---

## 二、用物理世界来理解

在传统工作里，你相信一份笔录是真的，通常因为：

```
✅ 有亲笔签名         ← 笔迹鉴定可以验证
✅ 有手印            ← 指纹鉴定可以验证
✅ 盖了单位公章        ← 公章是实物印章，别人仿制是犯罪
✅ 档案室归档了        ← 有专人管理，不能随便拿进去改
```

PrisonSIS 做的是：**把这些物理世界已有的保障，用数字手段延伸到电子文件上。**

---

## 三、数字指纹：怎么知道文件被改过

### 3.1 现实中的例子

你签完字、按完手印的笔录，放到档案室。

有一天有人说："这份笔录的第3页被人改过了。"

你怎么证明？

- 看纸质原件 → **纸质原件还在不在？**
- 如果原件还在，对比一下就知道有没有改 → **原件是可信的**

### 3.2 PrisonSIS 怎么做的

程序在上传扫描件时，会计算这份文件的**"数字指纹"**。

什么是数字指纹？就像人的指纹一样，每一个文件都有唯一的一串数字。

```
你的扫描件 PDF
      ↓
程序自动计算数字指纹（SHA-256）
      ↓
得到一串64位字符，例如：
e3b0c44298fc1c149afbf4c8996fb924...
      ↓
指纹被"焊死"在系统里
```

**如果有人偷偷改了这份文件，哪怕只改了一个字、一个标点，重新计算出的指纹都会完全不同。**

系统会自动比对：指纹对不上？说明被改过。系统立刻报警。

```
原件：数字指纹 = e3b0c44298fc1c...
改过：数字指纹 = a1b2c3d4e5f6...   ← 不一样！文件被篡改！
```

### 3.3 数字指纹能被伪造吗？

**不能。**

指纹的计算用的是一种叫 SHA-256 的数学算法，这是全世界公开验证过的标准。美国国家安全局（NSA）用它，中国政府也用它，银行系统用它，军队用它。

它的原理是：给定一个文件，可以算出指纹；但**给定指纹，不可能反推出原文件**；同时，**找不到两个不同的文件有相同的指纹**。

> 打个比方：你把一头大象碾成粉，告诉我这堆粉的颜色，我不可能用这堆粉重新造出一头大象来。

## 四、数字签名：怎么知道是谁上传的

### 4.1 现实中的例子

你有一枚私人印章，别人拿了你的章在文件上盖章，你说"那不是我盖的"。

印章丢了说不清楚。

### 4.2 PrisonSIS 的设计

PrisonSIS 不使用印章，而使用**钥匙**。

这套系统里，每个人有两把钥匙：

| 钥匙 | 在谁手里 | 用来做什么 |
|------|---------|----------|
| 公钥（公开钥匙） | 所有同事的电脑里都有 | 验证"这份文件是不是某人签的" |
| 私钥（保密钥匙） | **只有你自己知道** | 给文件签名 |

**私钥是程序在你首次登录时，自动生成并存在你自己电脑里的。**

私钥的样子是一串很长很长的乱码数字，比如：

```
-----BEGIN RSA PRIVATE KEY-----
MIIEowIBAAKCAQEA0Z3VS5K8N2P6QWx7H9jK2Mn4L5op8Q1TvW3rZ7XcY6A2...
[很长很长的乱码，这只是示意]
-----END RSA PRIVATE KEY-----
```

**这串东西只有你自己能看到，谁也看不到，包括程序员。**

### 4.3 签名是怎么工作的

当你上传扫描件时，程序做这件事：

```
你的扫描件 + 你的私钥 + 数字指纹
        ↓
程序用私钥对指纹做"签名"
        ↓
生成一份"签名凭证"（也是一串乱码数字）
        ↓
签名凭证和扫描件一起被保存
```

任何人拿到这份文件，都可以：
- 用你的公钥验证签名
- 确认"这个指纹是这个私钥签的"

但**只有持有私钥的人才能生成签名**。

### 4.4 最关键的一句话

> **私钥存在你自己的电脑里，程序员、管理员、任何人都看不到。你的 PIN 码只有你自己知道。程序没有能力伪造你的签名。**

这和印章不同——印章可以被偷走，可以被仿制。但私钥无法被偷走，无法被仿制，因为它的数学结构保证了：不知道就是不知道。

## 五、为什么连开发者都改不了

这是最核心的问题，很多人问：

> "你说改不了，万一程序员在代码里动手脚呢？"

### 5.1 先理解：代码能改什么，不能改什么

代码能做的是：
- ✅ 控制程序的行为（怎么计算指纹、怎么签名）
- ✅ 控制数据的存取方式

代码**做不到**的是：
- ❌ **绕过私钥签名** — 因为签名必须用私钥，而私钥在用户电脑里，不在代码里
- ❌ **算出你的私钥** — 私钥是随机生成的，代码无法预知
- ❌ **伪造数字指纹** — 指纹是根据文件内容实时计算的，代码改文件内容，指纹就会变

### 5.2 类比说明

程序员改代码，就像：
- 银行的保安系统设计师设计了一套门禁系统
- 设计师知道怎么设计门，但**不知道每个用户的密码**
- 用户进门必须输密码，设计师没办法绕过密码

在 PrisonSIS 里：
- 程序员知道怎么写"签名验证"的代码
- 但程序员**没有你的私钥，没有你的 PIN**
- 没有这两样东西，程序签不出你的名

### 5.3 技术上的保证

即使最坏的情况发生——程序员写了恶意代码、病毒感染了程序：

```
攻击者想做的事：偷偷改一份已签名的笔录

步骤1：改文件内容
结果：数字指纹立刻改变，和原签名对不上 → 系统发现被篡改

步骤2：重新计算指纹，伪造签名
前提：需要持有该用户的私钥
结果：私钥在用户电脑里，攻击者没有 → 伪造失败

步骤3：用程序本身的私钥签
结果：签名里记录了私钥持有人的身份
      验证方看到签名者不是原上传者 → 立刻发现签名被伪造
```

**三层防线：**
1. 改内容 → 指纹变了 → 立刻发现
2. 想绕过指纹 → 需要私钥 → 没有私钥
3. 用别的私钥签 → 签名者身份不对 → 立刻发现

### 5.4 最直接的保证

PrisonSIS 是**开源项目**，代码公开在 GitHub 上。

这意味着：
- 任何人都可以去查看代码，确认程序没有后门
- 可以让安全专家来审计代码
- 可以让上级主管部门来验证代码
- **如果有人改了代码，代码的变更记录会留在 GitHub 上，任何人都能看到**

这不是信任程序员的问题。这是**把安全建立在任何人都可以验证的基础上**。

## 七、关于纸质原件

最后要说明的是：

**PrisonSIS 不是要替代纸质档案，而是给纸质档案加了一层数字保护。**

纸质原件依然保存在档案室。

PrisonSIS 里的电子文件是备份和证明——它能证明：
- 这份笔录的纸质版是谁签的字
- 纸质原件的内容到现在有没有被改过
- 这份笔录是哪个时间、哪个民警上传的

万一纸质原件丢失或损坏，PrisonSIS 里的电子版本配合数字签名，依然可以作为强有力的法律证据。

## 八、去中心化存储的安全性

### 8.1 为什么扫描件可以用 P2P 网络传输

扫描件在上传时被 AES-256 加密，只有上传者持有解密私钥。

P2P 网络传输的是**加密后的密文文件**。即使有人在网络上截获了密文，或者其他节点的电脑上存储了这份密文，**没有私钥依然无法解密内容**。

```
原始 PDF 文件
      ↓
AES-256 加密（密钥随机生成）
      ↓
RSA-2048 加密 AES 密钥（AES密钥被上传者公钥包裹）
      ↓
JSON Bundle：{ encKey, iv, cipher }
      ↓
P2P 网络传输这个 JSON Bundle
      ↓
其他节点存储的是同样的密文 JSON Bundle
      ↓
只有持有私钥的上传者能解密 ✅
```

### 8.2 节点身份是如何验证的

节点身份不是用用户名或 IP 地址，而是用**公钥哈希**。

每个节点的 ID = SHA-256（该用户的 RSA 公钥 PEM）前 16 位字符。

这意味着：
- 不需要中央 CA 颁发证书
- 公钥哈希就代表了该用户，不需要额外的身份认证层
- 任何人可以声称自己是某个节点，但签名验证会揭穿谎言

### 8.3 节点离线或恶意节点

- **节点广播虚假 SCAN_INDEX**：签名验证会拒绝不属于自己的签名
- **恶意节点提供错误的扫描件密文**：接收方会校验 SHA-256 哈希，不匹配则丢弃
- **中间人攻击**：所有 TCP 连接在同一监狱内网，物理安全由内网保障
- **数据丢失**：多个节点同时存储同一份加密扫描件，丢失概率极低

### 8.4 与中央服务器方案的对比

| | 中央服务器存储 | PrisonSIS P2P 方案 |
|--|--------------|------------------|
| 服务器故障 | ❌ 笔录完全无法访问 | ✅ 任意节点可访问 |
| 数据篡改风险 | 服务器管理员可改 | ❌ 篡改后签名验证失败 |
| 私钥泄露风险 | 服务器管理员有数据 | ❌ 服务器只有密文，无私钥 |
| 单点故障 | ❌ 是 | ❌ 否 |
| 内网传输 | ❌ 有外部暴露风险 | ✅ 仅监狱内网 |
| 运维成本 | 高（需专人维护） | 低（节点即服务器）|

## 九、常见质疑的解答

**问：我的电脑中了病毒，私钥会不会被盗？**
答：有可能。所以请保管好你的电脑，定期更新杀毒软件。如果怀疑私钥泄露，请立即联系管理员重置你的密钥对。

**问：PIN 码忘了怎么办？**
答：PIN 码只能重置。但重置后，之前的数字签名将无法使用，之前签名的笔录仍然有效（因为签名本身已经存在，无法撤销）。建议将 PIN 码手写在纸上，放在安全的地方（不是电脑上）。

**问：如果有人拿着枪逼我签，这份笔录算自愿的吗？**
答：这个问题超出了技术能解决的范围。但 PrisonSIS 记录了签名的精确时间，如果时间异常（比如凌晨3点），可以作为辅助判断依据。

**问：万一硬盘坏了，笔录还在吗？**
答：如果其他同事在线，他们的电脑上有你草稿的备份（草稿部分）。正式签名的笔录因为是加密存储的，解密需要私钥，所以需要你保管好私钥备份（程序提供私钥导出功能，请将导出的文件备份到安全位置）。
