## 一、这个工具是什么

PrisonSIS，中文名叫**"审讯笔录助手"**，是一款专门为监狱民警制作的笔录管理工具。

你可以把它理解为**专门用来做笔录的"Word"**——但它比 Word 更聪明、更安全、更省心。

---

## 二、它能做什么

### 2.1 做笔录（核心功能）

在程序里填写审讯信息，选择笔录模板，直接开始编辑。系统会自动帮你排版、编号、保存。

### 2.2 打印笔录

编辑完成后，点击"打印"，程序会生成一份排版整齐的 PDF 文件，你只需要把它打印出来。

### 2.3 上传扫描件（关键步骤）

用扫描仪把签好字、按好手印的纸质版扫描成 PDF，上传到程序里。程序会自动对这份扫描件做"数字保护"——给它加密、计算"数字指纹"、加上你的数字签名，确保这份扫描件以后谁也改不了。

### 2.4 P2P 扫描件协作（去中心化）

不需要服务器，所有民警的电脑组成 P2P 网络。
上传扫描件后，签名信息会自动广播给所有在线节点；
其他工作站的同事可以随时从 P2P 网络下载、解密你上传的扫描件。

### 2.5 从其他工作站获取扫描件

如果需要在其他工作站查看某份笔录的扫描件：
1. 在笔录编辑器中点击**"📥 下载扫描件"**
2. 程序会显示 P2P 网络中各节点持有的扫描件列表
3. 选择对应笔录，输入 PIN 解密，即可导出 PDF 文件

## 三、为什么要用它，而不是直接用 Word

这是最重要的问题，我们直接对比：

### 3.1 排版问题

| | Word 做笔录 | PrisonSIS 做笔录 |
|---|---|---|
| 每次新建都要调格式 | ✅ 头疼 | ❌ 自动排好 |
| 打印出来格式乱掉 | ✅ 常有 | ❌ 不会 |
| 多人用同一模板 | ✅ 每次传文件 | ❌ 程序内置 |

> 你一定遇到过：用 Word 做的笔录，发给同事电脑打开，格式全乱了。PrisonSIS 的格式是程序保证的，任何电脑打开都一样。

### 3.2 保存问题

| | Word 做笔录 | PrisonSIS 做笔录 |
|---|---|---|
| 突然断电 | ✅ 白写了 | ❌ 自动保存草稿 |
| 服务器坏了 | ✅ 找不回来了 | ❌ 数据在本地 |
| 误删了文件 | ✅ 没救了 | ❌ 有草稿箱 |

> 用 Word 做笔录，最怕的就是：辛苦写了一上午，电脑蓝屏了，全没了。PrisonSIS 每 5 分钟自动保存一次草稿，不会丢。

### 3.3 证明笔录没被改过

| | Word 做笔录 | PrisonSIS 做笔录 |
|---|---|---|
| 证明内容没被改过 | ❌ 很难 | ✅ 数字指纹锁定 |
| 证明是谁写的 | ❌ 口头说 | ✅ 数字签名背书 |
| 纸质签名 + 数字保护 | ❌ 只有纸质 | ✅ 双重保障 |

> 这是最关键的区别。一份纸质笔录签字按手印后，用扫描仪扫成 PDF 上传到程序里，程序会自动做三件事：
> 1. **加密** — 这份文件只有你能看，别人看不到
> 2. **计算数字指纹** — 以后一旦文件被改过，系统能发现
> 3. **加上你的数字签名** — 证明这份文件是你上传的，谁也抵赖不了

### 3.4 总结：Word 能做笔录，但保护不了笔录

```
Word 做笔录 = 能写字
PrisonSIS 做笔录 = 能写字 + 自动保存 + 不会乱 + 打印规范 + 防篡改 + 同事共享
```

## 四、去中心化是什么意思

### 4.1 传统系统的样子

```
你的电脑 ←—————— 网络 ——————→ 服务器（某一台电脑）
                                    ↓
                               所有人依赖它
                               服务器坏了，全停了
                               数据在服务器上，丢了就没了
```

监狱的网络环境比较特殊，服务器故障并不罕见。一旦服务器挂了，谁也做不了笔录。

### 4.2 PrisonSIS 的样子

```
民警A的电脑（本地数据库）┐
民警B的电脑（本地数据库）┼── 局域网P2P ──→ 加密扫描件相互备份
民警C的电脑（本地数据库）┘
                          没有服务器
                          每台电脑都是节点
                          哪台坏了，其他都不受影响
```

**去中心化的核心意思：没有一台电脑是"必须的"。**

扫描件以上游加密密文形式在多个节点存储，任一节点均可独立解密验证。这在监狱这种需要严格留存记录的环境里，尤为重要。

### 4.3 监狱内网天然适合去中心化

监狱内网和互联网是隔离的，没有外部攻击者。在这样一个封闭的、安全的内网环境里，P2P 同步既快速（局域网速度）、又安全（隔离于外网）。这正是 PrisonSIS 选择去中心化架构的根本原因。

## 五、这份笔录，是真的吗？—— 怎么理解系统的可信度

这是大家最关心的问题：**我凭什么相信这份笔录没被改过？**

先记住三句话，然后再解释：

> **第一：纸质原件是根本，数字签名是保护。**
> **第二：谁也改不了，除非同时偷走你的电脑和你的 PIN。**
> **第三：代码是公开的，任何人都能验证。**

### 5.1 先说一个生活里的类比

你签字按手印的纸质版笔录，放进档案室。

有一天有人说"这份笔录第3页被人改过了"。

你怎么证明？

你拿出档案室的原件，一对比，就知道有没有改。原件在，就是真的。

PrisonSIS 做的是同一件事——**给电子版的扫描件，也生成一份"原件指纹"，焊死在系统里。**

以后任何人说"这份电子笔录被改过了"，系统立刻能验证：指纹对不对，对不上就是假的。

### 5.2 再说数字签名是怎么回事

签名用到一把钥匙，叫**私钥**。

这把钥匙是程序**在你登录时自动生成，存在你自己电脑里的**。

- **私钥只有你有**，程序员没有，管理员没有，任何人都没有
- **PIN 码只有你知道**，这是你自己设置的6位数字
- 上传扫描件时，程序用你的私钥 + 你的 PIN，给这份文件做签名
- 任何人以后都可以验证：这份文件是不是你签的

**私钥就相当于你的私人印章。** 纸质文件的印章可以被偷走仿制，但私钥在数学上无法被仿制——不知道就是不知道。

### 5.3 程序员能改我的笔录吗？

**改不了。**

就算程序员想动手脚，也做不到，因为：

- 改文件内容 → 数字指纹立刻变了 → 系统发现被篡改
- 想绕过指纹重新签名 → 需要私钥 → 程序员的代码里没有私钥
- 用别的钥匙签 → 签名人身份对不上 → 立刻露馅

### 5.4 代码公开可查

PrisonSIS 是**开源项目**，代码公开在互联网上，所有人都能查看。

这意味着：
- 安全专家可以验证程序没有后门
- 上级主管部门可以安排技术人员审查代码
- **不是信任程序员，而是把安全建立在任何人都可以验证的基础上**

### 5.5 简单总结

| 纸质档案靠什么可信 | PrisonSIS 对应的手段 |
|---|---|
| 亲笔签名（笔迹可鉴定） | 数字签名（数学可验证） |
| 手印（指纹可鉴定） | 数字指纹（算法保证唯一性） |
| 公章（实物印章难仿制） | 私钥（数学结构不可伪造） |
| 档案室专人管理 | 代码公开可查 |

**一句话总结：PrisonSIS 用数学手段，把纸质档案的信任体系延伸到了电子文件。**

详细内容请阅读配套的《安全性说明》文档。

## 五、核心概念：草稿和正式笔录

PrisonSIS 将笔录分为两个状态，理解这个区别非常重要：

### 5.1 草稿状态 📝

- **特点**：可以随意修改、删除、重新编辑
- **状态**：屏幕左上角显示"草稿"
- **用途**：还在填写、还没定稿的内容
- **同步**：会自动同步给局域网内所有在线的同事
- **法律效力**：无

> 类比：草稿就像写在笔记本上的草稿纸，还没定稿，怎么改都行。

### 5.2 正式笔录状态 🔒

- **特点**：已经打印、签字、扫描、上传，锁定不可更改
- **状态**：屏幕左上角显示"已签名扫描件"
- **用途**：经过所有签字按手印的正式笔录
- **同步**：P2P 网络会通知其他节点"这份笔录已签名"，但不传文件本身
- **法律效力**：有

> 类比：正式笔录就像已经提交给档案室的纸质文件，档案室不会让你回去涂改。

### 5.3 为什么这样设计

有人问：为什么不把正式笔录也让大家一起编辑？

**因为现实工作的逻辑就是这样的：**

```
写 → 打印 → 签字按手印 → 扫描上传 → 定稿
 ↑ 随意改         ↑ 谁也改不了
 草稿阶段               正式阶段
```

草稿阶段，你想怎么改都行，这不影响任何事。

但一旦签了字、扫描上传，这就是一份正式的、有法律效力的文件。任何对正式文件的篡改都是严重的违法行为。

**把这两个阶段分开，是让工具符合真实工作流程，而不是让真实工作流程去迁就工具。**

## 六、安全与隐私

### 6.1 我的数据安全吗？

**非常安全。** PrisonSIS 用了和银行、军队同等级别的加密技术：

| 保护措施 | 说明 |
|---------|------|
| AES-256 加密 | 军用级加密算法，你的扫描件只有你能解密 |
| SHA-256 数字指纹 | 文件一旦被改过，系统立刻能发现 |
| RSA-2048 数字签名 | 证明这份文件是你上传的，谁也抵赖不了 |
| 数据存在本地 | 不上传任何数据到外部服务器 |
| 监狱内网隔离 | 没有互联网攻击风险 |

### 6.2 别人能看到我的笔录吗？

**草稿**：局域网内在线同事可以看到，这是有意设计的，方便协作。
**正式扫描件**：只有你自己能解密，其他人的电脑只有记录"这份笔录已签名"，没有文件内容。

### 6.3 私钥是什么？我要保管密码吗？

私钥是程序自动为你生成的一把"数字钥匙"，它存在你自己的电脑里。

- 程序首次登录时会提示你设置一个 PIN 码，用来保护你的私钥
- 这个 PIN 码只有你知道，不会发给任何人
- 每次上传扫描件时，程序会要求输入这个 PIN 来"签名"
- **请务必记住这个 PIN，如果忘记将无法恢复**

## 七、常见问题

**Q：程序闪退或崩溃了，我的数据还在吗？**
A：在。程序每 5 分钟自动保存一次草稿，打开程序后可以在"草稿箱"里找到。

**Q：我电脑坏了，数据还能找回来吗？**
A：草稿保存在本地电脑，丢失后无法从其他节点恢复，请定期使用"数据备份"功能备份数据库。正式笔录的扫描件经过 AES-256 加密存储在其他节点，需要你的私钥（PIN）才能解密，请务必记住 PIN。

**Q：我忘了 PIN 码怎么办？**
A：很抱歉，如果忘记 PIN 码，将无法使用你的数字签名功能。届时请联系管理员重置。

**Q：内网断了，还能做笔录吗？**
A：完全可以。笔录编辑、保存都在本地完成，内网断开不影响正常工作。恢复连接后，草稿会自动同步给其他在线同事。

**Q：程序和同事的电脑不兼容，能用吗？**
A：PrisonSIS 基于 Qt6 开发，可在 Windows、Linux、macOS 上运行。只要在同一局域网内即可互相发现和同步。
